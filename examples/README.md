# SiMAL examples

This folder contains small, self-contained example schemas in **SiMAL** (`.siml`) plus the JSON artifacts generated from them.

If you just want to try SiMAL quickly, these examples are the easiest starting point.

## What’s in each example folder

Each example directory contains:

- `schema.siml` — the source SiMAL schema
- `schema.json` — **full** JSON (round-trippable; includes `__type__` tags)
- `schema_simple.json` — flattened, prompt-friendly JSON
- `schema_max_simple.json` — extra-compact JSON (keeps endpoints/methods as strings when possible)

The JSON files are generated by the CLI and can be regenerated at any time.

## Examples

### simple-ecommerce

A fictional ecommerce microservices context intended to demonstrate:

- rich `runtime` configs (dev/prod)
- `ci` pipelines
- service dependencies
- API groups (gRPC + HTTP)
- component blocks (`database`, `cache`, `struct`)
- LLM review feedback fields (`llm_feedback`)

Directory: `simple-ecommerce/`

### microservice-app-example

Сomplex polyglot microservice application model:

- Vue frontend calling multiple backend APIs
- Go Auth API issuing JWT tokens
- Node Todos API publishing messages to Redis
- Java Users API
- Python log message processor
- Zipkin tracing + Redis queue

Directory: `microservice-app-example/`

## Regenerate JSON outputs

From the repo root:

```bash
python simal_cli.py examples/microservice-app-example/schema.siml
python simal_cli.py examples/simple-ecommerce/schema.siml
```

To explicitly select outputs:

```bash
python simal_cli.py examples/simple-ecommerce/schema.siml --json
python simal_cli.py examples/simple-ecommerce/schema.siml --simple
python simal_cli.py examples/simple-ecommerce/schema.siml --max-simple
```
